Overall

The helix concept is genuinely distinguished: scroll-driven strand drawing with bloom/halo passes is rare and arresting. The node entrance sub-phase architecture is smart. But two problems undercut the premium feel: the node ring scales from absolute zero (robotic, not organic), and the hover card's exit is identically prominent to its enter (doors slamming both ways). Fix those two and the section reads as intentional craft. The 0.8s header entrance is also lagging, making the section feel sleepy before the helix can impress.

Critical Jakub

Node ring scales from scale(0) — mechanical, not organic

WhatThe ringScale sub-phase goes from 0 → 1 via transform: scale(${sub.ringScale}). Starting at true zero produces a robotic pop-in — the ring materializes from a singularity, not from space.

Why it mattersJakub's rule is clear: starting from scale(0) reads as "element was not here and is now here" — a mechanical binary. Starting from scale(0.6) reads as "element was small and grew" — a physical event. For a DNA node that is meant to feel like it coalesces out of the helix, organic growth from a non-zero starting point is essential.

Recommended motionChange ringScale: Math.min(1, np / 0.4) so the CSS transform starts at scale(0.55): transform: translate(-50%,-50%) scale(${0.55 + sub.ringScale * 0.45}). This gives a 0.55→1.0 growth arc that feels like crystallization rather than teleportation. Pair with cubic-bezier(0.22,1,0.36,1) on the parent element's transition instead of the linear scrub.

Critical Jakub

Card exit is as prominent as its enter — doors slamming both ways

WhatCard enter: opacity 0→1, blur(8px)→0, y: 12→0, duration 250ms. Card exit: opacity 1→0, filter: blur(4px), no y exit. The exit lacks a y-offset direction, and blur(4px) on exit is nearly as heavy as the blur(8px) enter. Result: leaving a node feels as dramatic as arriving.

Why it mattersJakub's exit rule is foundational: exits should be subtler — the user is moving on, the UI should step aside, not demand attention a second time. The missing y-exit means the card simply dissolves in place, breaking the spatial logic established by the y-enter.

Recommended motionExit: opacity: 0, filter: 'blur(2px)', y: -6, duration: 0.15. Half the blur, a short upward drift, and 60% of the enter duration. The card retreats upward (opposite of its entry direction) and disappears quickly. Change the current exit={{ opacity: 0, filter: 'blur(4px)' }} to exit={{ opacity: 0, filter: 'blur(2px)', y: -6 }} and transition: {{ duration: 0.15, ease: 'easeIn' }}.

Important Jhey

Orbital dot reveal uses CSS transition — kills stagger personality

WhatThe 30 orbital dots use transition: opacity ${0.1 + dotIdx * 0.015}s ease applied via inline style. This creates a stagger — but because it's a CSS transition driven by a boolean opacity (0 or 1), each dot either fully shows or fully hides. There is no scale, no blur, no size punch. The dots pop on sequentially with no physical feel.

Why it mattersFrom Jhey's lens: these dots are the section's only particle-like element. They have enormous delight potential — each dot appearing could have a micro-pop (scale 0→1.3→1, duration 120ms) that reads like sparks materializing along the ring. The current implementation wastes that moment on a bare opacity toggle.

Recommended motionReplace the inline CSS transition with a keyframe-driven appearance. For each dot with index dotIdx, apply: animation: dotPop 120ms cubic-bezier(0.34,1.56,0.64,1) forwards with animation-delay: ${dotIdx * 15}ms. Keyframe: 0% scale(0) opacity(0) → 60% scale(1.35) opacity(1) → 100% scale(1) opacity(1). The micro-overshoot on each dot landing gives the orbital ring a "magnetized" feel.

Important Emil

Header entrance at 0.8s — section's first impression moves in slow motion

WhatThe [data-reveal] elements animate at duration: 0.8, ease: 'expo.out'. Three elements stagger at 0ms, 80ms, and 160ms delays — the last element completes at 0.8 + 0.16 = 0.96s after the trigger. Nearly a full second of entrance before the helix can even register.

Why it mattersEven in a portfolio context where duration latitude is wider, 0.8s for a text entrance is on the edge. The expo.out easing means most of the motion happens in the first 40% of duration — the tail 60% is nearly imperceptible deceleration. This is 0.3s of wasted time. Emil's rule: if you can't feel the difference after the 80% mark, cut the duration.

Recommended motionReduce to duration: 0.5, ease: 'power3.out' and compress stagger delays to 0ms, 50ms, 100ms. Total reveal: 600ms. The expo.out can stay — just tighten the duration so the acceleration phase is the same but the invisible tail is removed. GSAP: gsap.to(el, { opacity: 1, y: 0, duration: 0.5, ease: 'power3.out', delay }).

Important Emil

prefers-reduced-motion: card blur transitions not suppressed in CSS

WhatThe @media (prefers-reduced-motion: reduce) block in the CSS only sets .nodeCard { transition: none; }. The Framer Motion card animations (initial, animate, exit, transition) are not governed by CSS — they still run at full intensity in reduced-motion mode because there is no useReducedMotion() hook in the component.

Why it mattersUsers with vestibular disorders who set prefers-reduced-motion get the full blur+y-offset hover card animation on every node interaction. The CSS override only targets static transition properties; Framer Motion's JS-driven animations are invisible to the CSS media query.

Recommended motionImport useReducedMotion from Framer Motion: const shouldReduceMotion = useReducedMotion(). In the card: initial={{ opacity: 0 }} (no blur, no y when reduced). animate={{ opacity: 1 }}. exit={{ opacity: 0 }}. Apply to children stagger too — zero y, zero delay when shouldReduceMotion is true.
